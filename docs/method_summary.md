# 方法探索记录

本文档用于记录项目过程中尝试过的训练思路和模块设计。部分策略属于实验探索或代码支持能力，不等同于最终提交中每一项都产生稳定收益。

## 1. 问题分析

网络监督细粒度图像识别的主要难点包括：

1. 网络爬取数据存在标签噪声；
2. 细粒度类别之间差异小，需要关注局部判别区域；
3. 训练集存在类别长尾分布；
4. 测试集为人工精确标注数据，要求模型具备较强泛化能力。

## 2. 总体方案

基础方案围绕单模型分类框架展开：

```text
Image → Data Augmentation → ConvNeXt Backbone → Classification Head → Class Prediction
```

训练阶段主要围绕噪声鲁棒学习、类别重加权、样本筛选和数据增强进行实验；推理阶段遵守单模型提交约束，输出比赛要求的 CSV 文件。

## 3. 主要实验方向

### 3.1 ConvNeXt Backbone

使用 ConvNeXt / ConvNeXtV2 作为主干网络，并限制为 ImageNet-1k 预训练权重，符合比赛约束。

### 3.2 WS-DAN 注意力机制

尝试在 backbone feature map 上预测注意力图，引导模型关注细粒度判别区域。训练时可对高响应区域进行 erase，促使模型学习互补局部特征。

### 3.3 ArcFace 分类头

尝试使用 margin-based 分类头扩大类间间隔，缓解细粒度类别相似导致的边界模糊问题。

### 3.4 噪声鲁棒损失

代码支持多种噪声鲁棒损失函数，用于对比不同训练配置下模型对错误标签的敏感性。

### 3.5 类别长尾处理

尝试采用 class-balanced sampler 和 class-aware loss reweighting，使尾部类别在训练中获得更充分的优化。

### 3.6 自清洗机制

利用预测置信度估计样本可靠性，对持续低置信样本降低训练权重，以减弱疑似噪声样本的影响。

### 3.7 伪标签机制

支持生成 soft pseudo label，并在后续训练中以较低权重混入训练集，进一步利用模型自身高置信预测。

## 4. 单模型约束说明

Uniform Model Soup 只对同一架构的多个 checkpoint 做权重平均，最终推理时仍加载一个 checkpoint，属于单模型推理，不是多模型 ensemble。
